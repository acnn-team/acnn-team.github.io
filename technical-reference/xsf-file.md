ACNN uses `.xsf` files as the labeled data format for training. In the active-learning workflow, DFT calculations are first completed in `DFT/`, then converted into `.xsf` files in `XSF/IT*/`. These `.xsf` files are copied into `POT/IT*/DT/` and used by the ACNN training step.

Use [`.res`](/technical-reference/res-file/) for structure pools, seed structures, and phase-diagram files. Use `.xsf` for training data that contains the structure together with DFT energy, virial, and forces.

## Where `.xsf` files appear

| Location | Meaning |
| --- | --- |
| `XSF/IT*/` | Labeled DFT data generated by `XSF/ry` and backend converters such as `vasp2xsf`. |
| `POT/IT*/DT/` | Training dataset copied from all available `XSF/IT*/` directories. |
| `PD/IT*/` | Phase-diagram scripts may convert current `.xsf` data back into `.res`-style structures for hull construction. |

## Minimal example

```console
# total energy = -21.42421364 eV

VIRIAL
         76.89614501        -63.24123555         -0.00002632        -63.24123555         72.93618155         -0.00002988         -0.00002632         -0.00002988         50.81682340
CRYSTAL
PRIMVEC
          4.99786000          0.00000000          0.00000000
         -0.15639807          4.99541232          0.00000000
         -2.42073105         -2.49770625          4.56496531
PRIMCOORD
3 1
Sr          0.67926000          4.18551000          0.00000000         -0.64865900          0.62866700         -0.00000800
B           1.28326000          3.60012000          1.60283000         -8.12401900          7.87363300          7.95771100
B          -1.13747000          1.10242000          2.96214000         -8.12400300          7.87365100         -7.95770700
```

## Required content

| Block | Meaning |
| --- | --- |
| `# total energy = ... eV` | Total DFT energy of the structure. |
| `VIRIAL` | 3 x 3 virial tensor written as nine values. This is an ACNN training extension, not a standard XCrySDen requirement. |
| `CRYSTAL` | Declares a periodic crystal structure. |
| `PRIMVEC` | Three primitive lattice vectors. |
| `PRIMCOORD` | Atom count followed by atom positions and forces. |

## Units

| Quantity | Unit |
| --- | --- |
| Lattice vectors in `PRIMVEC` | Å |
| Atomic positions in `PRIMCOORD` | Å |
| Total energy | eV |
| `VIRIAL` | eV |
| Forces | eV/Å in the normal ACNN DFT-converter convention |

The `VIRIAL` block is written as:

```console
VIRIAL
    xx  xy  xz  yx  yy  yz  zx  zy  zz
```

The virial values are used by ACNN training and are also important for high-pressure systems, where pressure-volume contributions affect structure ranking.

## `PRIMVEC`

```console
PRIMVEC
          4.99786000          0.00000000          0.00000000
         -0.15639807          4.99541232          0.00000000
         -2.42073105         -2.49770625          4.56496531
```

`PRIMVEC` stores the three lattice vectors in Cartesian coordinates. These vectors define the simulation cell used for the DFT calculation.

## `PRIMCOORD`

```console
PRIMCOORD
3 1
Sr          0.67926000          4.18551000          0.00000000         -0.64865900          0.62866700         -0.00000800
B           1.28326000          3.60012000          1.60283000         -8.12401900          7.87363300          7.95771100
B          -1.13747000          1.10242000          2.96214000         -8.12400300          7.87365100         -7.95770700
```

The first number after `PRIMCOORD` is the number of atoms, and it must match the number of atom lines that follow. The second number is the XCrySDen animation-step flag, normally written as `1` for a single structure. Each atom line contains:

```text
<element> <x> <y> <z> <fx> <fy> <fz>
```

The positions are Cartesian coordinates. The last three columns are force components from the DFT calculation.

## Sanity checks

After converting a DFT iteration into `.xsf` files, check the dataset before training:

```console
$ cd XSF
$ ./ry > log 2>&1
$ acnn_checkdt 50
```

Then confirm that the iteration directory is not empty:

```console
$ cd ..
$ find XSF/IT0 -name "*.xsf" | wc -l
```

An empty or corrupted `XSF/IT*/` directory will lead to an invalid ACNN training round, because `POT/tr` copies `.xsf` files from `XSF/*/` into the training dataset.
