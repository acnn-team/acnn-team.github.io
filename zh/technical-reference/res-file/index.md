ACNN 使用 SHELX 风格的 `.res` 文件作为紧凑的结构文件格式。在高压主动学习搜索中，`.res` 文件用于初始候选结构、DFT 优化后的 seed 结构、相图结构，以及每一轮 ACNN 优化后选入下一轮 DFT 的结构。

当你需要保存或传递晶体结构时，使用 `.res`。当你需要包含 DFT 能量、virial 和力的 ACNN 训练数据时，使用 [`.xsf`]({{ '/zh/technical-reference/xsf-file/' | relative_url }})。

## `.res` 文件出现的位置

| 位置 | 含义 |
| --- | --- |
| `RSS/Base/` | 第一轮 DFT 抽样使用的初始候选结构，可以是未优化结构。 |
| `SEED/` | DFT 优化过并带有计算能量的端点或参考结构，用于构建相图。 |
| `PD/IT*/` | 每一轮用于构建 convex hull 的结构。 |
| `RELAX/IT*/FPC/` | ACNN 优化后，被选入下一轮 DFT 标注的结构。 |

一个典型的 `.res` 文件包含：

| 行 | 含义 |
| --- | --- |
| `TITL` | 结构名和 ACNN/AIRSS 风格工具使用的元数据。 |
| `REM` | 注释信息。大多数 ACNN 结构解析不会依赖这些行。 |
| `CELL` | 晶胞参数。 |
| `LATT` | SHELX 记法中的点阵中心化和反演标记。 |
| `SFAC` | 元素列表，原子行中的元素编号对应该列表顺序。 |
| 原子行 | 元素标签、元素编号、分数坐标和占位数。 |
| `END` | 结构结束。 |

## `TITL` 元数据

ACNN 工具遵循常见的 AIRSS/SHELX 风格 `TITL` 约定：

```console
TITL SrB-2286444-4824-10   0     115.382       0       0        0      12      (C2)       n - 1
TITL <name> <pressure> <volume> <enthalpy> <spin> <modspin> <#ions> <(symmetry)> n - <#copies>
```

重要字段：

| 字段 | 含义 |
| --- | --- |
| `<name>` | 结构标识符，通常也作为文件名前缀。 |
| `<pressure>` | 压强，单位 GPa。未经过 DFT 标注的生成结构可能为 `0`。 |
| `<volume>` | 晶胞体积，单位 Å^3。 |
| `<enthalpy>` | 分析工具使用的焓或由能量得到的数值。生成结构可能为 `0`。 |
| `<#ions>` | 结构中的原子数。 |
| `<(symmetry)>` | 已知时给出空间群或对称性标签。 |
| `<#copies>` | 分析工具报告的等价或重复结构数量。 |

`RSS/Base/` 中未优化的随机结构，其能量相关字段可以是占位值。`SEED/` 中的 seed 结构应来自 DFT 优化后的计算结果，否则相图没有物理意义。

## `CELL`

```console
CELL 1.54180    4.95734    4.95734    5.07324   84.16089   84.16089   68.83962
```

第一个数是 SHELX 的 wavelength 字段，通常保留为 `1.54180`。后面六个数是晶胞参数：

```text
CELL <wavelength> <a> <b> <c> <alpha> <beta> <gamma>
```

长度单位为 Å，角度单位为度。

## `SFAC` 和原子行

```console
SFAC Sr B
Sr     1  0.5654957000000  0.5112210000000  0.7154375000000 1.0
B      2  0.5408928000000  0.8437647000000  0.1371216000000 1.0
```

`SFAC` 定义元素顺序。原子行中元素标签后的整数是 `SFAC` 中的元素编号：

| 值 | 含义 |
| --- | --- |
| 原子标签 | 通常是化学元素符号，如 `Sr` 或 `B`。 |
| 元素编号 | `1` 表示 `SFAC` 中第一个元素，`2` 表示第二个元素，依此类推。 |
| `x y z` | 分数坐标。 |
| 占位数 | ACNN 结构搜索流程中通常为 `1.0`。 |

## 常用转换和检查

需要在 `.res` 和其他结构格式之间转换时，可以使用 `cabal`：

```console
$ cabal res poscar < input.res > POSCAR
$ cabal poscar res < POSCAR > output.res
```

可以使用 `ca` 或相关分析工具检查结构标识、组分、体积、对称性和焓等元数据。当 `.res` 文件作为 seed 结构使用时，务必确认它是 DFT 优化后的结构，并且组分和压强正确。
