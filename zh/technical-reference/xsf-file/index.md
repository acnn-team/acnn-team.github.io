ACNN 使用 `.xsf` 文件作为训练数据格式。在主动学习流程中，DFT 计算先在 `DFT/` 中完成，然后转换为 `XSF/IT*/` 中的 `.xsf` 文件。这些 `.xsf` 文件会被复制到 `POT/IT*/DT/`，并用于 ACNN 势函数训练。

结构池、seed 结构和相图文件使用 [`.res`](/zh/technical-reference/res-file/)。包含 DFT 能量、virial 和力的训练数据使用 `.xsf`。

## `.xsf` 文件出现的位置

| 位置 | 含义 |
| --- | --- |
| `XSF/IT*/` | 由 `XSF/ry` 和 `vasp2xsf` 等后端转换器生成的 DFT 标注数据。 |
| `POT/IT*/DT/` | 从所有可用 `XSF/IT*/` 目录复制得到的训练数据集。 |
| `PD/IT*/` | 相图脚本可能会把当前 `.xsf` 数据转换回 `.res` 风格结构，用于 hull 构建。 |

CRYSTAL
PRIMVEC
          4.99786000          0.00000000          0.00000000
         -0.15639807          4.99541232          0.00000000
         -2.42073105         -2.49770625          4.56496531
PRIMCOORD
3 1
Sr          0.67926000          4.18551000          0.00000000         -0.64865900          0.62866700         -0.00000800
B           1.28326000          3.60012000          1.60283000         -8.12401900          7.87363300          7.95771100
B          -1.13747000          1.10242000          2.96214000         -8.12400300          7.87365100         -7.95770700

## 必要内容

| 块 | 含义 |
| --- | --- |
| `# total energy = ... eV` | 结构的 DFT 总能。 |
| `VIRIAL` | 以 9 个数写出的 3 x 3 virial 张量。这是 ACNN 训练使用的扩展字段，不是标准 XCrySDen 的必需字段。 |
| `CRYSTAL` | 声明周期性晶体结构。 |
| `PRIMVEC` | 三个原胞晶格矢量。 |
| `PRIMCOORD` | 原子数，以及原子坐标和力。 |

## 单位

| 物理量 | 单位 |
| --- | --- |
| `PRIMVEC` 中的晶格矢量 | Å |
| `PRIMCOORD` 中的原子坐标 | Å |
| 总能 | eV |
| `VIRIAL` | eV |
| 力 | 正常 ACNN DFT 转换约定中为 eV/Å |

`VIRIAL` 块写作：

```console
VIRIAL
    xx  xy  xz  yx  yy  yz  zx  zy  zz
```

virial 数值用于 ACNN 训练。在高压体系中，压力-体积项会影响结构排序，因此这些信息很重要。

## `PRIMVEC`

```console
PRIMVEC
          4.99786000          0.00000000          0.00000000
         -0.15639807          4.99541232          0.00000000
         -2.42073105         -2.49770625          4.56496531
```

`PRIMVEC` 以笛卡尔坐标保存三个晶格矢量，它们定义 DFT 计算使用的模拟晶胞。

## `PRIMCOORD`

```console
PRIMCOORD
3 1
Sr          0.67926000          4.18551000          0.00000000         -0.64865900          0.62866700         -0.00000800
B           1.28326000          3.60012000          1.60283000         -8.12401900          7.87363300          7.95771100
B          -1.13747000          1.10242000          2.96214000         -8.12400300          7.87365100         -7.95770700
```

`PRIMCOORD` 后的第一个数字是原子数，必须与后面实际给出的原子行数一致。第二个数字是 XCrySDen 的动画步标记，单个结构通常写作 `1`。每一行原子信息包含：

```text
<element> <x> <y> <z> <fx> <fy> <fz>
```

坐标是笛卡尔坐标。最后三列是 DFT 计算得到的力分量。

## 检查数据

把一轮 DFT 结果转换成 `.xsf` 文件后，训练前应检查数据集：

```console
$ cd XSF
$ ./ry > log 2>&1
$ acnn_checkdt 50
```

然后确认这一轮目录不是空的：

```console
$ cd ..
$ find XSF/IT0 -name "*.xsf" | wc -l
```

如果 `XSF/IT*/` 为空或文件损坏，会导致 ACNN 训练轮次无效，因为 `POT/tr` 会从 `XSF/*/` 复制 `.xsf` 文件到训练数据集中。
